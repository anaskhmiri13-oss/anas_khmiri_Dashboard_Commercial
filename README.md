* Ce projet consiste à développer un dashboard avec un style moderne permettant de visualiser différentes données commerciales : les ventes, les revenus et les produits.
Il inclut plusieurs sections interactives, des graphiques , un affichage des produits, ainsi que des animations légères pour une meilleur utilisation .
*Comme technologie j ai utilise HTML pour la structure de la page , CSS pour la mise en forme , design moderne ainsi que pour les animation , js pour l interaction et la gestion des donnes dnas le dashboard ,
charts.js une bibliotheque  de js qui cree des graphiques de bonnes qualites et finalment github pour les commits .
*les fonctionnalités principales de mon site son : le menu interactif qui permet la navigations entre les sections , graphique des ventes (line chart) ,graphique des revenus (bar chart) , graphique des prix 
des produits (doughnut chart) , tableau dynamique listant les produits et tous ces parties sont equipes d une legere animation , une signature dans le header et l adaptation mobile du site . 
* Durant ce projet, j’ai découver plusieurs nouvelles notions surtout en rapport avec le js comme la manipulation de chart.js pour generer pluqieurs types de graphiques , 
le gestion du dom pour l affichage des produits avec simplicites , les animations des graphiques ,tableau , sections grace au css et finalment l utilisation du git et github  .
* Comme difficultés, j’ai rencontré plusieurs problèmes : la taille des charts, l’affichage dynamique du tableau, l’organisation des fichiers du projet, comment faire des animations et quel type utiliser. 
J’ai aussi voulu faire un bouton de toggle pour passer du mode dark au mode light, mais j’ai échoué du côté du code, donc j’ai retiré l’idée. J’ai fait beaucoup de tests afin que ce site soit comme il est aujourd’hui
* Afin de résoudre ces problèmes, j’ai utilisé plusieurs méthodes : la recherche dans la documentation officielle de Chart.js, l’utilisation de .chart-container pour l’ajustement des charts, 
le test de plusieurs animations possibles afin de trouver la meilleure et la plus simple, l’amélioration progressive du design pour obtenir ce rendu propre, le partage du code en cinq grandes parties pour 
être plus clair pour moi et pour les autres, ainsi que la recherche dans d’autres sites web pour m’inspirer, comme pour la signature, la palette de couleurs utilisée ou le style du menu de navigation.
